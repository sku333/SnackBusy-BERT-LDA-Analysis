# 硬折扣零食店评论分析 — 深度学习方法升级方案

## 一、原始分析回顾 (论文第三章)

### 1.1 数据来源
- **平台**: 大众点评、美团
- **品牌**: 零食很忙、好想来
- **数据量**: 1377 条用户评论 (CSV 格式, GBK 编码, 单列 "评论内容")
- **清洗后**: 1375 条有效评论 (情感分析) / 1372 条 (LDA 主题建模)

### 1.2 原有分析流程
```
采集 → 清洗 → Jieba分词 → SnowNLP情感分析 → LDA主题建模 → 词云图
```

### 1.3 原有结果摘要

| 环节 | 方法 | 结果 |
|------|------|------|
| 情感分析 | SnowNLP | 正向 73.8%, 中性 3.9%, 负向 22.3% |
| 好评主题1 | LDA | 价格, 便宜, 性价比, 方便, 不错 |
| 好评主题2 | LDA | 环境, 服务, 品种, 实惠, 整洁 |
| 好评主题3 | LDA | 好吃, 口味, 品类, 丰富, 味道 |
| 差评主题1 | LDA | 收银, 结账, 排队, 收银员, 袋子 |
| 差评主题2 | LDA | 服务, 态度, 商品, 品牌, 营业员 |
| 差评主题3 | LDA | 会员, 价格, 便宜, 货架, 品种 |
| 词云 | Jieba+WordCloud | 正向/负向 两张词云图 |

## 二、升级目标

在不更改原始数据的前提下，用更先进的深度学习模型重跑全流程，
以便在论文中形成"传统方法 vs 深度学习方法"的对比论证。

## 三、升级方法设计

### 3.1 情感分析升级: SnowNLP → RoBERTa

```
原方法: SnowNLP (基于朴素贝叶斯的规则模型)
新方法: uer/roberta-base-finetuned-jd-binary-chinese
```

**选型理由:**
- 该模型在京东电商评论上做过微调 (fine-tuning)，与我们的零食店评论领域高度相关
- 基于 RoBERTa 架构，对中文语义的理解远超 SnowNLP
- 输出二分类 (positive/negative) + 置信度分数

**注意事项:**
- 该模型为二分类，无"中性"类别。原 3.9% 的中性评论将被分配至正/负向
- 推理在 CPU 上完成，batch_size=16，约需 2-3 分钟

### 3.2 主题挖掘升级: Jieba+LDA → BERT+KMeans+LDA

```
原方法: Jieba分词 → 词袋模型 → gensim LDA
新方法: Sentence-BERT 句子向量 → KMeans 聚类 → sklearn LDA 关键词提取
```

**流程详解:**

Step 1 — 句子向量化: 使用 `paraphrase-multilingual-MiniLM-L12-v2` 将每条评论
    映射为 384 维语义向量。该模型支持 50+ 种语言，在中文语义相似度任务上
    表现优异。

Step 2 — 语义聚类: 对正/负评论分别使用 KMeans (k=3) 进行聚类。与传统 LDA
    基于词共现的聚类不同，KMeans 在语义空间中进行聚类，能更好地捕捉
    语义层面的主题结构。

Step 3 — 关键词提取: 对每个聚类内的评论文本，使用 sklearn 的 LDA 提取
    Top-5 关键词，作为该聚类的"主题标签"。

**优势:**
- BERT 提供深层语义理解，KMeans 在语义空间聚类
- 相比纯词频统计的 LDA，能发现更丰富的主题结构
- 最终仍输出关键词列表，便于与原结果直接对比

### 3.3 词云升级: Jieba → BertTokenizer

```
原方法: Jieba 分词 → WordCloud
新方法: RoBERTa BertTokenizer (BPE) → WordCloud
```

**说明:**
- 使用情感分析模型自带的 BertTokenizer，基于 BPE (Byte-Pair Encoding) 算法切词
- 子词级别的分析粒度，更贴近模型内部的文本表示方式
- 作为 Jieba 的对比方案，展示不同分词策略对词云的影响

## 五、技术保障

### 5.1 鲁棒性设计
- **编码自动检测**: 依次尝试 gbk, gb18030, utf-8-sig, utf-8
- **空文本处理**: 清洗阶段过滤空值、纯符号、模板评论
- **模型加载重试**: 网络异常时自动重试 3 次，间隔 5 秒
- **Token 回退**: BertTokenizer 无法产生有效 token 时自动回退到 Jieba

### 5.2 环境适配
- **HF 镜像**: 默认使用 hf-mirror.com，解决 huggingface.co 不可达问题
- **本地加载**: 支持 `--local` 参数加载预先下载的模型
- **CPU 推理**: 全流程不依赖 GPU，适合服务器环境
- **字体自动查找**: 在 Linux/Windows/macOS 上自动搜索中文字体

### 5.3 可复现性
- 所有随机种子固定为 42
- 依赖版本在 requirements.txt 中明确指定
- 输出结果包含时间戳，支持多次运行对比

## 六、执行步骤

```
1. 安装 Python 环境 (Python 3.8+)
2. pip install -r requirements.txt
3. 安装中文字体 (Linux: apt install fonts-wqy-zenhei)
4. python main.py                    # 自动从 HF 镜像下载模型并运行
   或
   python main.py --local            # 使用本地已下载的模型
5. 查看 output/ 目录下的所有分析结果
```

| 模型 | 用途 | 大小 | HuggingFace 路径 |
|------|------|------|------------------|
| RoBERTa | 情感分析 | ~400MB | uer/roberta-base-finetuned-jd-binary-chinese |
